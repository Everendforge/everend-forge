# Monetization

Everend Forge should keep the open source core useful and complete. Monetization should focus on convenience, support, services, and optional hosted capabilities rather than locking core formats.

## Recommended first paths

- Paid signed installers and managed releases.
- Priority support.
- Consulting and studio onboarding.
- Premium templates and optional asset packs.
- GitHub Sponsors, Patreon, or Ko-fi.

## Later options

- Optional cloud sync, backup, collaboration, or hosted previews.
- Dual licensing for studios only if runtime embedding creates real commercial demand.

## Default stance

Do not lock the spec, basic authoring workflows, or runtime package execution behind proprietary access.
